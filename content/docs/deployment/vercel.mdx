# Deploy to Vercel

Deploy your Get Cracked SaaS application to Vercel with this comprehensive guide.

## Prerequisites

Before deploying, ensure you have:

- A Vercel account (free to create)
- Your Get Cracked codebase with all dependencies installed
- Environment variables configured
- Database set up and running

## Step 1: Prepare Your Codebase

### Environment Variables

Create a `.env.local` file in your project root with all required environment variables:

```env
# Database
DATABASE_URL="your-database-url"

# Clerk Authentication
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="your-clerk-publishable-key"
CLERK_SECRET_KEY="your-clerk-secret-key"
NEXT_PUBLIC_CLERK_SIGN_IN_URL="/sign-in"
NEXT_PUBLIC_CLERK_SIGN_UP_URL="/sign-up"

# Stripe
STRIPE_SECRET_KEY="your-stripe-secret-key"
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY="your-stripe-publishable-key"
STRIPE_WEBHOOK_SECRET="your-stripe-webhook-secret"

# Resend Email
RESEND_API_KEY="your-resend-api-key"

# App Configuration
NEXT_PUBLIC_APP_URL="https://your-domain.vercel.app"
```

Never commit your `.env.local` file to version control. Add it to your `.gitignore`.

### Database Setup

Ensure your database is accessible from the internet:

1. **Neon Database**: Already cloud-hosted and accessible
2. **Supabase**: Use the connection string from your project settings
3. **Railway**: Enable public networking in your database settings
4. **PlanetScale**: Use the public connection string

Test your database connection:

```bash
pnpm run db:push
```

## Step 2: Deploy to Vercel

### Using Vercel CLI

1. **Install Vercel CLI**:
```bash
npm i -g vercel
```

2. **Login to Vercel**:
```bash
vercel login
```

3. **Deploy**:
```bash
vercel --prod
```

### Using GitHub Integration

1. **Push to GitHub**: Push your code to a GitHub repository
2. **Connect to Vercel**: Go to [vercel.com](https://vercel.com) and import your repository
3. **Configure Settings**: Vercel will automatically detect Next.js settings
4. **Deploy**: Click "Deploy" to start the deployment process

## Step 3: Configure Environment Variables

In the Vercel dashboard:

1. Go to your project settings
2. Navigate to **Environment Variables**
3. Add all the variables from your `.env.local` file
4. Redeploy your application

Use the same variable names in Vercel as in your local `.env.local` file.

## Step 4: Set Up Database

1. **Run Migrations**: Vercel will automatically run `pnpm run build`
2. **Check Database**: Ensure your database is accessible and migrations run successfully
3. **Verify Connection**: Check that your app can connect to the database

## Step 5: Configure Authentication

### Clerk Setup

1. **Domain Configuration**: Add your Vercel domain to Clerk allowed domains
2. **Webhook URLs**: Update webhook URLs in Clerk dashboard:
   - User created: `https://your-domain.vercel.app/api/webhooks/clerk/user-created`
   - User updated: `https://your-domain.vercel.app/api/webhooks/clerk/user-updated`

3. **Environment Variables**: Ensure Clerk environment variables are set in Vercel

## Step 6: Configure Payments

### Stripe Setup

1. **Webhook Configuration**: Add your Vercel domain to Stripe webhook endpoints
2. **Webhook Events**: Configure these events:
   - `customer.subscription.created`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
   - `invoice.payment_succeeded`
   - `invoice.payment_failed`

3. **Webhook URL**: `https://your-domain.vercel.app/api/webhooks/stripe`

## Step 7: Configure Email

### Resend Setup

1. **Domain Verification**: Verify your domain in Resend dashboard
2. **Environment Variables**: Ensure Resend API key is configured in Vercel
3. **Test Email**: Send a test email to verify delivery

## Troubleshooting

### Common Issues

#### Database Connection Errors

**Problem**: App can't connect to database
**Solution**:
- Verify DATABASE_URL is correct in Vercel
- Check database is accessible from internet
- Ensure database credentials are valid

#### Build Failures

**Problem**: Deployment fails during build
**Solution**:
- Check all environment variables are set
- Verify Node.js version compatibility
- Review build logs for specific errors

#### Authentication Issues

**Problem**: Clerk authentication not working
**Solution**:
- Verify Clerk environment variables
- Check domain is added to Clerk settings
- Ensure HTTPS is enabled (required by Clerk)

#### Stripe Webhook Failures

**Problem**: Stripe webhooks not processing
**Solution**:
- Verify webhook URL is correct
- Check webhook secret matches
- Ensure endpoint can receive POST requests
